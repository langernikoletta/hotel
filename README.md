# Hotel

## Що було виправлено
- Виправлено помилку імпорту в `urls.py` та `main_views.py` (повернуті пакетні імпорти).
- Проєкт налаштовано як пакет: `INSTALLED_APPS` містить `hotel`, оновлено `ROOT_URLCONF` та `WSGI_APPLICATION`.
- `DJANGO_SETTINGS_MODULE` переведено на `hotel.settings` у `manage.py` та `wsgi.py`, додано шлях до кореня проєкту.
- `base.html` переміщено в `templates/`, щоб Django знаходив шаблон.

## Короткий фідбек по структурі
- Плюс: мінімалістична структура та зрозумілий поділ `templates/main/*`.
- Мінус: немає окремого Django‑app (наприклад `main/`) — зараз уся логіка в пакеті `hotel`. Це працює для навчального проєкту, але для росту краще виділити app.
- Рекомендація: винести `models.py`, `views.py`, `urls.py` в окремий app і підключити його в `INSTALLED_APPS`.
